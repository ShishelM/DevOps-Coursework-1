# Курсовая работа на профессии "`DevOps-инженер с нуля`" - `Рыбянцев Павел`

Данный проект реализует отказоустойчивую инфраструктуру с централизованным сбором метрик и логов. Все внутренние сервисы изолированы в приватных подсетях и доступны через единую точку входа.
## 🏗 Архитектура решения

* Сеть: VPC с публичными и приватными подсетями в двух зонах доступности (ru-central1-a, ru-central1-b).
* Точка входа: Application Load Balancer (ALB), распределяющий трафик на группу Web-серверов.
* Web-серверы: Два инстанса Nginx, выполняющие роль сайта и Reverse Proxy для Grafana и Kibana.
* Мониторинг: Prometheus (сбор метрик) + Grafana (визуализация).
* Логирование: ELK Stack (Elasticsearch + Kibana), сбор данных через Filebeat с JSON-парсингом.
* База данных: Managed PostgreSQL для долгосрочного хранения данных.*
* Безопасность: Доступ к приватным узлам только через Bastion-хост по SSH. Публичный доступ к UI мониторинга ограничен через Nginx Proxy.
* Бэкапы: Автоматические снимки дисков (Snapshot Schedule) по расписанию.

 `* Передача метрик и адаптер  для записи закомментирован, так как для успешного завершения требуется установка extension для PostgreSQL (для работы с временными рядами)`

## 🛠 Стек технологий

* IaC: Terraform
* Конфигурация: Ansible
* Контейнеризация: Docker
* Облако: Yandex Cloud

## 🚀 Быстрый старт## 1. Подготовка облака
Убедитесь, что у вас установлен yc CLI, создан профиль и настроены переменные в variables.tf (folder_id, cloud_id, токен).
## 2. Развертывание инфраструктуры

```
terraform init
terraform apply -auto-approve
```
`Terraform создаст сеть, виртуалки, базу данных и сгенерирует актуальный hosts.ini для Ansible.`

## 3. Конфигурация сервисов

```
ansible-playbook site.yml
```

## 📈 Ссылки на сервисы
После успешного деплоя все сервисы доступны по адресу Балансировщика (ALB IP):

* Сайт: http://<ALB_IP>/
* Grafana: http://<ALB_IP>/grafana/ (admin/admin)
* Kibana: http://<ALB_IP>/kibana/

## 💡 Особенности реализации

   1. Zero Public IP: Сервисы Grafana, Kibana, Prometheus и DB не имеют публичных IP-адресов. Доступ к веб-интерфейсам реализован через proxy_pass в Nginx.
   2. JSON Logging: Nginx настроен на генерацию логов в формате JSON. Filebeat собирает их и отправляет в Elasticsearch, что позволяет строить детальную аналитику в Kibana без сложного парсинга строк.
   3. DB Integration: Реализована сетевая связность с Managed PostgreSQL.*
   4. Оптимизация: Ресурсы ВМ подобраны с учетом стабильной работы Java-приложений (Elasticsearch) при минимальных затратах облачного бюджета.
   
   `* Передача метрик и адаптер  для записи закомментирован, так как для успешного завершения требуется установка extension для PostgreSQL (для работы с временными рядами)`
